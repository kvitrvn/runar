## Skills
A skill is a set of local instructions stored in a `SKILL.md` file. Use the available skills below when the user explicitly names one or when the task clearly matches the skill description.

### Available skills
- expert-senior-go-developer: Senior Go engineering workflow for designing, implementing, refactoring, debugging, profiling, and reviewing Go services, CLIs, libraries, workers, and APIs. Use when work in this repo needs a high bar on package design, interfaces, error handling, concurrency, context propagation, testing, performance, compatibility, or operational safety. (file: `.codex/skills/expert-senior-go-developer/SKILL.md`)
- expert-juridique-auto-entreprise: French legal and compliance workflow for auto-entrepreneur and micro-entreprise questions. Use when work needs current official guidance on status, obligations, thresholds, registration, invoicing rules, VAT treatment, social and tax regime, declarations, activity changes, cessation, or risk points for a French auto-entrepreneur. (file: `.codex/skills/expert-juridique-auto-entreprise/SKILL.md`)

### How to use skills
- Open the `SKILL.md` only after deciding the skill applies.
- Read only the specific bundled files needed for the task.
- If multiple skills apply, use the minimum set and state the order briefly.
- If a skill file cannot be read, continue with the best fallback and mention the issue briefly.
